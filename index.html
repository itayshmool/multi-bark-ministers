A named minister-pup is spawned or resumed in a tmux session. Netanyahu, Gallant, Lapid — whoever is next in rotation (or whomever you @mention by name). The minister reads the system prompt, reviews session history, and gets to work. No cabinet meeting required.

Government Ministries · המשרדים

THE MINISTRIES OF STATE

Each Ministry oversees a critical domain of the AI Government. All ministers serve at the pleasure of the user and may be replaced at any time via /clear pack.

🏛️
MINISTRY NO. 1

MINISTRY OF BACKENDS

Oversees four LLM engines: Claude Code, Cursor, OpenAI Codex, and Google Gemini. Switch backends with a hashtag. The ministry guarantees no vendor lock-in, only geopolitical lock-in.

📜
MINISTRY NO. 2

MINISTRY OF MEMORY

Ensures persistent conversation sessions across server restarts. Each pup remembers its full history. Rolling summaries keep context lean. Amnesia is not a policy option.

🎙️
MINISTRY NO. 3

MINISTRY OF COMMUNICATIONS

Voice messages are transcribed locally via whisper.cpp — no cloud required. Supports Hebrew and 90 other languages. The minister hears all. The minister transcribes all.

⚖️
MINISTRY NO. 4

MINISTRY OF JUSTICE

Auto-recovery orchestrates fallback across three modes: retry, reset, and backend-switch. Context window overflow, rate limits, and crashes are met with the full force of jurisprudence. No pup left behind.

📁
MINISTRY NO. 5

MINISTRY OF INTERIOR

Manages file delivery directly to chat. Pups drop files into their outbox; the minister personally delivers them via sendFile(). PDFs, PNGs, ZIPs — no format is beneath the ministry's notice.

🤝
MINISTRY NO. 6

MINISTRY OF FOREIGN AFFAIRS

Governs pup delegation. Senior pups may spawn up to 3 sub-agents via bark delegate. Sub-agents work independently, appear in chat, and file PRs autonomously. Diplomacy is encouraged. Nested sessions are not.

Government Decrees · גזרות הממשלה

THE OFFICIAL GOVERNMENT DECREES

The following decrees have been signed into law by the Prime Minister Pup. All citizens are expected to comply. Violations are handled by the Ministry of Justice (auto-recovery).

DECREE № 1
/status

The Prime Minister orders an immediate status refresh. All active pups, their backends, and efficiency ratings are displayed. Coalition stability confirmed.

DECREE № 2
/stop <name>

The named minister is sent a polite Ctrl+C. Work ceases. The pup remains in position, available for resumption. No severance package.

DECREE № 3
/clear <name>

The minister is shelved — deactivated but not forgotten. Session history is preserved. Reborn at a moment's notice. Use pack to shelve the entire cabinet.

DECREE № 4
/reborn <name>

A shelved minister is resurrected with full memory intact. The pup returns exactly as it was, carrying the weight of its entire session history.

DECREE № 5
/reset <name>

Ministerial amnesia is invoked. The pup starts a fresh session while remaining active. Same backend, same position, clean slate. No memory of past indiscretions.

DECREE № 6
/daily

The Prime Minister requests a one-line standup from every active pup simultaneously. The people deserve transparency. Ministers respond within seconds.

DECREE № 7
/stats [name]

Token usage, cost, and efficiency per-minister are displayed. The finance ministry demands accountability. Every API call is logged and accounted for.

DECREE № 8
/create

Reply to any message to spawn a new pup with that message as context. Optionally add @name to appoint a specific minister. Government grows by popular demand.

DECREE № 9
/backends

A full roll-call of coalition parties is displayed with capabilities and version numbers. Coalition composition is public record. No secrets from the people.

Coalition Formation · הרכבת הקואליציה

HOW TO JOIN THE COALITION

Coalition formation follows a strict four-article process. Prerequisites must be installed. No compromise on this point. The Prime Minister insists.

ARTICLE I · PREREQUISITES

Install the infrastructure

Node.js, tmux, Claude CLI, ffmpeg, and whisper.cpp. The government runs on prerequisites. Run the official installer:

# Clone the repository git clone https://github.com/itayshmool/multi-bark-pack cd multi-bark-pack # Run the prerequisite installer bash prerequisites.sh # Install all LLM backends and authenticate bash install-backends.sh
ARTICLE II · CONFIGURATION

Draft the coalition agreement

Create your .env file. This is the binding coalition agreement. Every signatory platform must be configured:

# .env — The Coalition Agreement DEFAULT_BACKEND=claude-code ENABLED_BACKENDS=claude-code,cursor,codex,gemini # WhatsApp adapter WA_ENABLED=true WA_GROUP=bark-pack # Or Telegram TELEGRAM_TOKEN=your-bot-token # Or Slack SLACK_BOT_TOKEN=xoxb-... SLACK_APP_TOKEN=xapp-...
ARTICLE III · MINISTER PACK

Assign ministerial names (optional)

The Israeli Ministers pack uses custom naming. Pups are named Netanyahu, Gallant, Lapid, Smotrich, Ben-Gvir, and more. Names cycle automatically. The first 32 are bare names, then adjective combos:

# Pup naming is automatic — first spawn gets "Netanyahu" # Next gets "Gallant", then "Lapid", and so on... # Once all names are used, combo names activate: # "Persistent-Netanyahu", "Bold-Gallant", etc. # Or @mention a specific minister by name: @Netanyahu please write a test for auth.js
ARTICLE IV · SWEARING IN

Start the government

Launch the server. The government auto-restarts on clean exit. Scan the QR code for WhatsApp. Ministers are on standby:

# Start with auto-restart (recommended) yarn start # Or without auto-restart node server.js # Monitor a minister's session tmux attach -t bark-Netanyahu # See all active ministers tmux ls
